# API 연동 이슈 정리 (9) — UI/UX 점검 8번 항목 수정 중 발견

> `docs/ui-ux-audit.md`의 "8. 검색 / 챗봇 / 404"를 고치면서 확인한 것. 8-1(검색결과 없음 안내), 8-3(404
> 페이지)은 순수 프론트 작업이라 백엔드와 무관하다. 8-2만 실제로 없는 기능이라 남긴다.

## 1. 검색창 카메라 아이콘("이미지 검색")은 백엔드에 대응 기능 자체가 없음

- **한 것**: 아무 동작도 없던 카메라 아이콘을 버튼으로 감싸서 "이미지 검색은 준비 중인 기능입니다" 토스트를
  띄우도록 최소한의 피드백만 추가했다.
- **왜 진짜로 구현 안 했나**: `src/api/search.ts`를 확인해보면 `getSuggestions`(자동완성)와
  검색기록 CRUD만 있고, 이미지를 업로드해서 유사 상품을 찾는 엔드포인트는 없다. 이건 UI 문구/아이콘 문제가
  아니라 이미지 검색 자체가 백엔드에 없는 기능이라, 이번 스코프(UI/UX 다듬기)로는 채울 수 없었다.
- **필요한 것**: 실제로 이미지 검색을 만들려면 이미지 업로드 API + 이미지 기반 상품 매칭 로직이 새로
  필요하다. 별도 기능 개발 건으로 분리하는 걸 권장.
