# 앱팡 UI/UX 점검 리포트

> 기준: 쿠팡 스타일의 전체적인 톤앤매너/레이아웃은 유지하되, 사용성(UX)이나 완성도 측면에서 문제가 되는 지점을 찾아 정리.
> 범위: `src/pages/*`, `src/components/*` 전체 17개 페이지.
> 이 문서는 문제 진단과 개선 방향 제안만 담고 있으며, 코드 수정은 포함하지 않음.

---

## 요약: 우선순위 Top 5

| 순위 | 이슈 | 영향 범위 |
|---|---|---|
| 1 | 상품 이미지가 대부분 화면에서 회색 박스로만 표시됨 | 상품목록, 장바구니, 주문내역, 주문상세, 마이페이지 |
| 2 | "쿠팡"/"쿠페이" 브랜드 텍스트가 그대로 남아있음 | 마이페이지, 상품목록 정렬 옵션 |
| 3 | 클릭해도 아무 반응 없는 "죽은 버튼"이 다수 존재 | 상품상세, 로그인, 회원가입, 마이페이지, 검색 등 |
| 4 | 주소 입력이 순수 텍스트 한 줄로만 처리되어 우편번호 검색 기능이 없음 | 배송지 등록/수정 |
| 5 | 옵션(색상/수량) 선택 UI가 실제 구매 동작과 연결되지 않은 장식용 UI | 상품상세 |

## 1. 전역 공통 이슈

### 1-1. 상품 이미지가 거의 항상 회색 박스로 표시됨
- **위치**: `ProductListPage.tsx:95`, `CartPage.tsx:183`, `OrderListPage.tsx:52`, `OrderDetailPage.tsx:109`, `MyPage.tsx`(OrderCard, 79행)
- **문제**: 실제 `<img>` 태그 없이 전부 `<div className="bg-gray-200" />` 로 고정되어 있음. 반면 `ProductDetailPage.tsx`는 `product.images` 배열로 실제 이미지를 렌더링하고 있어, 백엔드에 이미지 데이터가 있다는 걸 알 수 있음.
- **왜 문제인가**: 커머스 앱에서 상품 이미지는 구매 결정에 가장 큰 영향을 주는 요소인데, 목록/장바구니/주문내역 어디서도 사진이 안 보이면 사용자가 "이게 진짜 뭘 산 건지" 확인할 방법이 없음. 쿠팡 스타일의 톤은 유지했지만 정작 쿠팡의 핵심 경험(썸네일 스캔)이 빠져 있는 상태.
- **개선 제안**: 상품상세와 동일하게 각 리스트/카드 컴포넌트에 실제 이미지 URL을 `<img>`로 렌더링. 이미지가 없는 경우에만 회색 박스를 fallback으로 사용.

### 1-2. "쿠팡"/"쿠페이" 브랜드 텍스트가 그대로 남아있음
- **위치**: `MyPage.tsx:142` (`쿠팡 캐시`), `MyPage.tsx:146` (`쿠페이 머니`), `ProductListPage.tsx:29` (`쿠팡 랭킹순`)
- **문제**: 디자인 레퍼런스를 그대로 베끼는 과정에서 브랜드명이 "앱팡"으로 치환되지 않고 남음.
- **왜 문제인가**: 사용자가 보기에 매우 눈에 띄는 실수이고, 실제 서비스라면 상표권 문제로도 이어질 수 있음. 지금 상태로는 QA를 거치지 않은 느낌을 강하게 줌.
- **개선 제안**: `쿠팡 캐시` → `앱팡 캐시`(또는 기획에 맞는 명칭), `쿠페이 머니` → `앱팡 머니`, `쿠팡 랭킹순` → `앱팡 랭킹순` 등으로 전수 교체. grep으로 "쿠팡"/"쿠페이" 전체 재검색 권장.

### 1-3. 클릭해도 반응 없는 "죽은 버튼"이 여러 화면에 흩어져 있음
- **위치(대표)**:
  - `ProductDetailPage.tsx`: 공유하기(274행), 브랜드명 클릭(283행), "모든 옵션 보기"(390행), 리뷰 "전체보기"(417행), "리뷰 전체보기"(484행), 포토그리드 "더보기" 타일(471행, 버튼도 아님)
  - `LoginPage.tsx`: "아이디·비밀번호 찾기"(125행), "사업자 회원 가입하기"(146행)
  - `RegisterPage.tsx`: 각 약관의 "상세보기" 화살표(354행) — 필수 동의 항목인데 내용을 확인할 방법이 없음
  - `SearchPage.tsx`: 검색창 우측 카메라 아이콘(199행, 이미지 검색으로 보이나 미연결)
  - `MyPage.tsx`: 주문 카드 전체(OrderCard, 69행) 탭해도 상세로 이동 안 함, "고객센터"(231행)
- **왜 문제인가**: 화살표(`>`)나 버튼 스타일은 "누르면 뭔가 일어난다"는 강한 시각적 신호인데 실제로는 아무 일도 안 일어나면, 사용자는 앱이 고장났다고 느끼거나 신뢰를 잃음. 특히 회원가입 약관 "상세보기"는 법적으로도 약관 내용을 확인시켜야 하는 항목이라 더 중요함.
- **개선 제안**: 백엔드/기획이 아직 없는 기능은 (a) 버튼을 숨기거나 (b) "준비 중입니다" 토스트라도 붙여서 최소한의 피드백을 주는 것을 권장. 특히 마이페이지 주문 카드는 탭 시 상세 이동을 붙이는 게 가장 시급함(이미 옆에 있는 "전체보기"와 기능이 겹치므로 사용자 기대와 어긋남).

## 2. 메인 페이지 (MainPage)

### 2-1. 카테고리 아이콘과 라벨이 서로 안 맞음
- **위치**: `MainPage.tsx:25-86` (CATEGORIES 배열)
- **문제**: `생활용품` → 팝콘 아이콘, `뷰티` → 우유 아이콘, `출산·유아` → 의류 아이콘(의류·잡화와 중복), `스포츠·레저`/`자동차용품` → 둘 다 로켓 아이콘(중복).
- **왜 문제인가**: 아이콘은 텍스트를 읽기 전에 먼저 인식되는 요소인데, 의미가 안 맞으면 오히려 탐색 속도를 늦추고 "대충 만든 느낌"을 줌. 두 카테고리가 같은 아이콘을 쓰는 것도 구분력을 떨어뜨림.
- **개선 제안**: 카테고리별 의미가 맞는 아이콘으로 교체 (예: 생활용품→세제/솔, 뷰티→립스틱/거울, 자동차용품→자동차, 스포츠→덤벨/공 등). lucide-react 안에서도 더 적합한 아이콘을 찾을 수 있음.

### 2-2. 배너 슬라이더가 터치 스와이프에만 반응
- **위치**: `MainPage.tsx:141-179`
- **문제**: `onTouchStart/onTouchEnd`만 구현되어 있어 마우스 드래그(데스크톱 브라우저)로는 배너를 넘길 수 없음. 도트를 눌러야만 이동 가능.
- **왜 문제인가**: 모바일 전용이라면 문제 없지만, 웹으로 접근하는 사용자가 있다면 배너 인터랙션이 절반만 동작하는 것처럼 느껴짐.
- **개선 제안**: 모바일 전용 앱이라면 무시 가능. 반응형 웹까지 고려한다면 마우스 드래그 핸들러 또는 좌우 화살표 버튼 추가.

## 3. 상품 목록 / 상품 상세

### 3-1. 옵션(색상/수량) 선택이 실제 구매에 반영되지 않음
- **위치**: `ProductDetailPage.tsx:50-56`(COLOR_OPTIONS/QUANTITY_OPTIONS), `165-175`(handleAddToCart), `177-192`(handleBuyNow)
- **문제**: 사용자가 "60개" 옵션을 선택해도 장바구니 담기/바로구매는 항상 `quantity: 1`로 고정 전송됨. 색상 선택도 저장되지 않음.
- **왜 문제인가**: UI가 "선택했다"는 강한 신호(테두리 강조, 체크 표시)를 주는데 실제로는 무시되는 상태. 사용자가 60개를 주문했다고 생각했는데 1개만 왔다면 신뢰가 크게 깨짐. (코드 주석에 백엔드 옵션 API가 없어 데모용이라 명시되어 있어, 현재는 의도된 임시 상태로 보임)
- **개선 제안**: 백엔드 옵션 API가 준비되기 전까지는 수량 옵션을 선택 가능한 것처럼 보이지 않게 하거나(비활성/안내 문구), 최소한 선택한 수량만큼은 실제 quantity에 반영.

### 3-2. "리뷰 작성하기" 버튼이 왜 비활성화됐는지 안내가 없음
- **위치**: `ProductDetailPage.tsx:438-449`
- **문제**: `disabled={isLoggedIn && !canWriteReview}` 조건으로 회색 처리만 되고, 왜 못 쓰는지(구매 이력 없음 등) 설명이 없음.
- **개선 제안**: 비활성 상태에 툴팁이나 헬프텍스트로 "구매 후 작성 가능합니다" 같은 안내 추가.

### 3-3. 로딩/에러 화면에 `min-h-screen`, 본문에는 `h-screen` 혼용
- **위치**: `ProductDetailPage.tsx:198, 206` vs `217`
- **왜 문제인가**: 이 프로젝트는 standalone 페이지에서 `h-screen + flex-1 overflow-y-auto` 컨벤션을 쓰기로 되어 있는데, 로딩/에러 분기만 `min-h-screen`을 써서 레이아웃 셸이 페이지 상태에 따라 달라짐. 실기기에서 주소창 표시/키보드 노출 시 로딩 화면만 스크롤이 생기거나 레이아웃이 살짝 흔들릴 수 있음.
- **개선 제안**: 로딩/에러 분기도 동일한 셸(`h-screen`, 헤더 포함) 안에서 렌더링하도록 통일.

### 3-4. 상품 목록에 필터가 전혀 없음
- **위치**: `ProductListPage.tsx` 정렬바(244-276행)
- **문제**: 정렬(4종)만 있고 가격대/브랜드/배송유형 등 필터가 없음.
- **개선 제안**: 스코프에 따라 우선순위는 낮지만, 쿠팡 스타일을 표방한다면 최소 "로켓배송만 보기" 토글 정도는 기대되는 요소.

## 4. 장바구니 / 결제 / 주문

### 4-1. 장바구니 선택삭제 실패 시 롤백이 없음
- **위치**: `CartPage.tsx:309-322` (confirmDelete)
- **문제**: 화면에서는 즉시 아이템을 지우고, `deleteCartItem(id).catch(() => {})`로 실패를 그냥 무시함.
- **왜 문제인가**: 서버 삭제가 실패해도 사용자는 성공한 것으로 보고, 다음에 장바구니를 다시 열면 지운 줄 알았던 상품이 되살아나 있어 혼란을 줌.
- **개선 제안**: 실패 시 아이템을 되돌리고 에러 토스트 표시 (수량 변경 로직인 `changeQty`는 이미 이 패턴으로 잘 구현돼 있어 동일하게 맞추면 됨).

### 4-2. 결제수단이 선택 가능한 것처럼 보이지만 실제로는 1개뿐
- **위치**: `CheckoutPage.tsx:190-195`
- **문제**: 라디오 버튼 형태(원 안에 점)로 표시되지만 클릭 핸들러가 없고 선택지도 "앱팡 머니" 하나뿐.
- **왜 문제인가**: 라디오 UI는 "다른 것도 고를 수 있다"는 신호인데 실제로는 고정값이라 사용자가 혼란을 느낌.
- **개선 제안**: 결제수단이 하나뿐이라면 라디오 대신 단순 텍스트로 표시하거나, 추후 다른 결제수단이 생길 걸 대비한 것이라면 지금은 "선택 가능" 느낌을 빼는 게 맞음.

### 4-3. 결제 진행 중 피드백 부족
- **위치**: `CheckoutPage.tsx:230-238`
- **문제**: `isSubmitting` 동안 버튼은 disabled 되지만 텍스트가 "결제하기"에서 바뀌지 않아 로딩 스피너/문구가 없음.
- **개선 제안**: 다른 화면(로그인, 회원가입)처럼 스피너로 교체하거나 "결제 처리 중..." 문구 표시.

### 4-4. "배송 N건 중 N" 문구가 의미 없이 항상 같은 숫자를 반복
- **위치**: `CheckoutPage.tsx:148-150`
- **문제**: `배송 {items.length}건 중 {items.length}` — 분리배송 등의 실제 분기 로직이 없어 앞뒤 숫자가 항상 동일함.
- **개선 제안**: 분리배송 개념이 없다면 이 문구 자체를 제거하거나 "총 N개 상품" 같은 의미 있는 표현으로 교체.

### 4-5. 주문내역에서 같은 날짜의 여러 주문이 시각적으로 구분되지 않음
- **위치**: `OrderListPage.tsx:114-120` (날짜별 그룹핑)
- **문제**: 날짜로만 묶고 주문 단위 구분선/카드가 없어서, 하루에 주문을 두 번 했다면 한 주문처럼 보임.
- **개선 제안**: 같은 날짜 안에서도 주문 단위로 그룹 배경/여백을 줘서 구분.

### 4-6. "바로 구매" 재주문 시 수량이 항상 1로 초기화됨
- **위치**: `OrderListPage.tsx:107-112`
- **문제**: 원래 3개를 주문했던 상품도 "바로 구매"를 누르면 수량 1로 결제 페이지에 들어감.
- **개선 제안**: 원래 주문 수량을 유지하거나, 최소한 결제 페이지 진입 후 바로 수량 조절이 눈에 띄게 안내.

### 4-7. 주문상세에 리뷰 작성 / 교환·반품 같은 후속 액션이 없음
- **위치**: `OrderDetailPage.tsx` 전체
- **문제**: 결제 정보/배송지/상품 목록만 있고, 쿠팡류 앱에서 기대되는 "리뷰 쓰기", "교환/반품 신청", "배송조회" 버튼이 없음.
- **개선 제안**: 백엔드 지원 여부에 따라 우선순위 조정 필요하지만, 최소 리뷰 작성 버튼은 상품상세 페이지와 연결해 넣는 걸 권장.

## 5. 배송지 관리

### 5-1. 주소 입력이 텍스트 한 줄, 우편번호 검색 없음
- **위치**: `AddressFormPage.tsx:288-293`, `248`행(zipcode 항상 빈 문자열)
- **문제**: "주소를 입력하세요" 인풋 하나에 사용자가 도로명주소를 통째로 직접 타이핑해야 함. `zipcode`는 저장 시 항상 빈 문자열로 전송됨.
- **왜 문제인가**: 국내 커머스/배송 앱에서는 다음(Daum) 우편번호 검색 등으로 주소를 탭 한 번에 채우는 게 표준 경험임. 직접 타이핑은 오탈자 위험이 크고, 특히 배송지는 오탈자가 곧 배송 실패로 이어지는 민감한 데이터라 UX 임팩트가 큼.
- **개선 제안**: 우편번호 검색 팝업(주소 API) 연동 후 도로명주소 + 상세주소 입력을 분리. 최소한 지금처럼 하나로 합쳐서 저장하지 말고 상세주소 필드를 별도로 받기.

### 5-2. "일반배송"과 "로켓/새벽배송" 용어가 화면마다 다름
- **위치**: `AddressListPage.tsx:24-25`(`로켓 :`) vs `AddressFormPage.tsx:313`(`새벽 배송 정보`)
- **문제**: 같은 개념을 목록 화면은 "로켓", 입력 화면은 "새벽배송"으로 표기.
- **개선 제안**: 한 가지 용어로 통일.

### 5-3. 배송지 삭제 기능이 없음
- **위치**: `AddressListPage.tsx`, `AddressFormPage.tsx` 전체
- **문제**: 등록/수정만 가능하고 삭제 버튼이 어디에도 없음.
- **개선 제안**: 목록 카드 또는 수정 화면에 삭제 액션 추가(실수 방지를 위한 확인 모달은 장바구니 삭제 패턴 재사용 가능).

### 5-4. 배송 요청사항 선택 후 자동으로 접히지 않음
- **위치**: `AddressFormPage.tsx:96-195` (DeliverySection)
- **문제**: 옵션을 선택해도 아코디언이 계속 펼쳐진 채로 남아 사용자가 직접 헤더를 다시 눌러 접어야 함.
- **개선 제안**: 라디오 선택 시 자동으로 접어서 선택 요약이 보이게 하면 입력 흐름이 더 매끄러움.

## 6. 로그인 / 회원가입 / 설정

### 6-1. 로그인 실패 토스트는 수동으로만 닫히고, 다른 토스트는 자동으로 사라짐
- **위치**: `LoginPage.tsx:152-162` vs `CartPage.tsx`/`SettingsPage.tsx`의 2초 자동 소멸 토스트
- **문제**: 앱 전체에서 토스트 동작 방식이 통일돼 있지 않음.
- **개선 제안**: 공통 Toast 컴포넌트로 분리해 자동 소멸 시간/닫기 버튼 유무를 통일.

### 6-2. 설정 페이지에서 이름/이메일/전화번호 수정 시 기존 값이 입력창에 채워지지 않음
- **위치**: `SettingsPage.tsx:161-174` (toggleEditName 등, input을 매번 빈 문자열로 리셋)
- **문제**: "수정" 버튼을 누르면 빈 입력창이 뜨기 때문에 기존 이름 일부만 바꾸고 싶어도 전체를 처음부터 다시 입력해야 함.
- **개선 제안**: 편집 모드 진입 시 현재 값으로 입력창을 초기화.

### 6-3. 회원가입 약관 "상세보기"를 눌러도 내용을 볼 수 없음 (6-1과 중복이지만 회원가입 화면 관점에서 재정리)
- **위치**: `RegisterPage.tsx:353-357`
- **개선 제안**: 필수 동의 항목이므로 최소 모달/새 창으로 약관 전문을 확인할 수 있게 연결 필요.

### 6-4. 비밀번호 변경 화면에는 비밀번호 표시(눈 아이콘) 토글이 없음
- **위치**: `SettingsPage.tsx:302-320`
- **문제**: 로그인/회원가입 화면은 눈 아이콘으로 평문 확인이 가능한데, 설정의 비밀번호 변경 3개 필드는 항상 마스킹.
- **개선 제안**: 동일한 패턴으로 통일해 오타 확인이 가능하게 함.

## 7. 마이페이지

### 7-1. 주문 카드가 탭해도 반응 없음 + 로켓뱃지가 항상 표시 안 됨
- **위치**: `MyPage.tsx:32-38`(`hasRocket: false` 하드코딩), `67-86`(OrderCard, onClick 없음)
- **문제**: 주문 카드에 로켓 배송 뱃지를 그릴 조건문은 있지만 값이 항상 false라 절대 렌더링되지 않는 죽은 코드. 카드 자체도 클릭 핸들러가 없어 탭해도 아무 반응이 없음.
- **개선 제안**: `hasRocket`을 실제 주문 데이터에서 매핑하거나, 아직 데이터가 없다면 뱃지 자체를 제거. 카드 클릭 시 주문상세로 이동하는 핸들러 추가.

### 7-2. 주문 카드 우측 하단의 원형 아이콘 의미가 불분명
- **위치**: `MyPage.tsx:80-82` (PackagePlus 아이콘)
- **문제**: 라벨도 없고 클릭도 안 되는 아이콘이 썸네일 위에 떠 있어 용도를 짐작하기 어려움.
- **개선 제안**: "재구매" 등 의도된 기능이면 버튼으로 만들고 라벨/aria-label을 부여, 의미 없는 장식이면 제거.
